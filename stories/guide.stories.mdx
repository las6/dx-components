# dx-components

Copy-paste-ready DX components for the Vite + TanStack ecosystem â€” filling gaps that Next.js handles natively.

## ErrorComponent

Dev-friendly error overlay with parseable stack frames, chained `cause` display, copy-to-clipboard, and own-code line highlighting. Auto-detects `import.meta.env.PROD` and delegates to `ErrorPage` in production.

- **`mode: "dev"`** (default) â€” full dark overlay with call stack and chained errors
- **`mode: "prod"`** â€” delegates to `ErrorPage` for a clean user-facing display

<div style={{ background: "#0d0d0d", minHeight: "60dvh", padding: "1rem" }}>
  <ErrorComponent
    error={new Error("Could not load user data â€” database connection refused")}
    reset={() => {}}
    subtitle="Loader or server function threw"
  />
</div>

**Props:** `error` (unknown), `reset` (() => void), `subtitle?` (string), `mode?` ("dev" | "prod")

## Usage

### TanStack Router (SPA)

Use directly as `errorComponent`, `notFoundComponent`, or `pendingComponent`:

```tsx
import { createRootRoute } from "@tanstack/react-router";
import { ErrorComponent } from "./components/ErrorComponent";

export const Route = createRootRoute({
  errorComponent: ErrorComponent,
});
```

### TanStack Start (SSR)

The **root route** requires an HTML shell wrapper for `errorComponent` and `notFoundComponent`:

```tsx
import { createRootRoute, HeadContent, Scripts } from "@tanstack/react-router";
import { ErrorComponent } from "./components/ErrorComponent";

function RootErrorShell(props: ErrorComponentProps) {
  return (
    <html lang="en">
      <head><HeadContent /></head>
      <body style={{ margin: 0, background: "#0d0d0d" }}>
        <ErrorComponent {...props} />
        <Scripts />
      </body>
    </html>
  );
}

export const Route = createRootRoute({
  errorComponent: RootErrorShell,
});
```

`pendingComponent` never needs an HTML shell â€” it renders inside the parent layout.

## Error Reporting

Integrate an error tracking SDK (e.g. [Centry](https://github.com/las6/centry-client)) by wrapping `errorComponent`:

```tsx
import { useEffect } from "react";
import { captureException } from "centry-client";
import { ErrorComponent } from "./components/ErrorComponent";

function RootErrorComponent(props: ErrorComponentProps) {
  useEffect(() => {
    captureException(props.error);
  }, [props.error]);

  return <ErrorComponent {...props} />;
}
```

Initialize centry once at app startup:

```ts
import { init, installGlobalHandlers } from "centry-client";

const centryClient = init({
  project: "my-project",
  enabled: import.meta.env.PROD,
});
installGlobalHandlers(centryClient);
```

## Theming

All components use scoped `<style>` blocks with CSS custom properties. Override any variable on a parent wrapper or `:root`:

| Variable | Purpose | Dark default |
|---|---|---|
| `--dx-bg` | Page background | `#0d0d0d` |
| `--dx-text` | Primary text | `#e8e8e8` |
| `--dx-muted` | Secondary text | `#888` |
| `--dx-accent` | Error/accent | `#c00` |
| `--dx-surface` | Card background | `#161616` |
| `--dx-border` | Border color | `#2a2a2a` |
| `--dx-font` | Font family | `"Geist Mono", monospace` |

Use the preset switcher (top-right of the story canvas) to preview themes: **Dark**, **Light**, **Minimal**, **Inherit**.

To force a theme, copy the preset block into your global CSS or a parent wrapper. Components auto-detect `prefers-color-scheme: light` for their defaults.

## Design

- **Inline styles** â€” zero dependencies, no Tailwind or CSS imports
- **Dark theme** â€” monospace font stack, `100dvh` min-height
- **Self-contained** â€” each file is a single `.tsx` with its own `<style>` block
- **No build step** â€” ships as source; your bundler handles TypeScript + JSX
